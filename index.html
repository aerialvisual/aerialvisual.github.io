    HTML and CSS Help Reference Sheet

    Common Terms and Basic Guide

    HTML
    <head>

    Page Data is covered in the HTML <head> tag. In text editors like Atom, there is a built in "tab" function to automatically add your basic "boilderplate" code by typing "html" and hitting the tab key. The <head> tag stores the <meta>, the <link> tags, and the <title> tag.

    <header>

    The <header> is at the top of a webpage. It is inside of the <body> and typically is the parent element of the <nav> element. The <header> is often set to position: fixed in CSS. Using a reset.css is important because the header will render and space slightly differently than you intend as a developer without it.

    <link>

    The <link> HTML tag is, again, nested in the <header>. It has three sections with many argument choices. Syntax for the arguments is: <link href="url" type="texttype" rel="argument" />. <link> tags are used to link CSS sheets, online CDNs (ex. Bootstrap), and external APIs (Like Google Fonts).

    <nav> and <li>

    <nav> elements are usually nested in <header> or <footer> elements, but can be anywhere on a page. A <nav> <li> is usually wrapped in an <a> (link) tag. Using CSS to transform the basic list really turns a static <nav> list into a good looking navigation menu bar. The bar at the top of the page is exactly that; a fixed <header> with a <nav> styled with CSS.

    <ul>, <ol>, and <li>

    <ul> tags are unordered lists, with <li> child elements. In other words, bullet lists.

    <ol> tags are ordered with <li> child elements. These are numbered.

    <li> elements are inside of other tags and are the individual list items.

    <h1> thru <h6>

    <h1> thru <h6> tags define headers (different than the <header> tag). These tags usually do NOT render when a reset.css is in place, so they are treated like a <span> or <div> tag that you can just set font-size: value; in CSS.

    <span>

    <span> elements are one-line versions of the <div> grouping element. These elements don't alter content or size; they are there to help you organize and place elements using CSS, CSS Grid, CSS Flexbox, etc. They allow you to select text or words inbetween their opening and closing lines and assign a class, ID, or property directly. Every tag (<tag>) on this site that you can read is rendered with a <span> element, like <<span>exampletag<span>>. The extra <> on the end allows the browser to display the tag even though it really wants to treat it as non-text code.

    <div>

    <div> grouping elements are prehaps the most important HTML organizational element. By using logic and writing code in a readable, understandable way, web developers use <div> tags to group other divs, create rows and columns, organize content, style content, position elements, and basically lay out the page however they see fit.

    <p>

    <p> elements are just your ordinary paragraph, longer text tags. They are usually nested in <divs> and have multiple lines. Often, the styling <strong> or <em> tags are used to affect text, and CSS classes are applied for text manipulation.

    <strong>

    <strong> tags are very simple. Text that they surround are bold. If you are using a reset.css you will have to declare somewhere in your style.css that <strong> has a font-weight of bold. You could even add this CSS call to reset.css and it will take effect once again.

    <em>

    <em> tags similar to <strong> tags. They make the font-style of text they surround italic. Again, like <strong> tags, you will have to declare <em> to have a font-style of italic in reset.css or style.css for this to take effect because reset.css negates these calls.

    <a>

    The <a> tag is a link. No, not a <link>, but a literal link that you can direct to local or remote pages. Syntax for the <a> tag is: <a href="url" target="_blank">. The target attribute if not set opens the link in the same window, leaving your site. More often than not, it is best to set the target to "_blank" so your site is still open in another tab.

    <u>

    The <u> tag underlines text that the opening and closing brakets surround. Underlined text looks like this. Unline <strong> and <em>, reset.css should not cancel the effect of the <u> tag.

    <title>

    The <title> tag is in the <head> element. What you put between <title> and </title> is what the browser displays as your website's name in the tab. Shorter titles are optimal since there isn't much space for more than three or four words. For example, this website's <title> value is "Coding Reference".

    <meta>

    <meta> tags are on one line and can be self-closing. It is advisable to write your <meta> tag like this: <meta charset="utf-8" name="viewport" content="width=device-width, initial-scale=1"> The first argument in the above example generates with the boilerplate code but adding the other three ensures that the browser will render your webpage's sizing based on the size of the screen it's viewed on. Without this added, there are often issues when writing the mobile version of a website.

    HTML Information

    Organization

    When writing HTML, it is important to stay organized. HTML is a logical language that can be very confusing if not properly organized. Always use indentation to indicate parent and child <div> elements. Make sure your code is readable, because when you are writing a lot of code, it's hard to fix a specific issue if you struggle to find the actual code.

    Troubleshooting

    Troubleshooting your code will be very important as you develop as a web designer. Use tools like Google, Mozilla Documentation, Dev Tools, and StackOverflow (to name a few) to perfect your website.

    The power of the <div>

    The <div> tag is again, an important organizational element. It allows you to organize your code in a logical manner, using CSS classes (bootstrap not required) to style. I extensively use CSS Flexbox to organize and layout my webpages. This whole webpage is made of parent <div> tags with child <div> tags and CSS Flexbox to put it all where I want.

    CSS

    font-family

    The font-family property of CSS is able to be set to a number of pre-defined fonts, as well as imported fonts from somewhere like Google Fonts.

    Some generic system-safe fonts are:
    • - serif or sans-serif
    • - monospace
    • - cursive
    • - fantasy

    Proper Usage of the font-family property:

      selector {
         font-family: "Imported font", font;
    }

    font-weight

    The font-weight CSS property allows you to easily bold or manually set text to a weight, between 100 and 900. 400 is considered "normal" font weight, 300 is "light" and 700 is "bold".

    font-weight takes the following values:
    • - normal
    • - bold
    • - lighter
    • - bolder
    • - Number between 100 and 900

    Proper Usage of the font-weight property:

      selector {
         font-weight: bold;
    }

    font-style

    The font-style property is used with values like italic.

    The values for font-style are:
    • - italic
    • - oblique
    • - oblique (deg)

    font-style is useful especially when using an imported font that either doesn't have an italic typeface or you didn't import it. It can manually apply a slant to text for you, and you can customize how slanted the text is. If your typeface has italic characters, setting it to oblique will just make it italic, as below.
    Italic Text     Oblique Text

    Proper Usage of the font-style property:

      selector {
         font-style: oblique 30deg;
    }

    text-align

    The text-align CSS property is very important when laying out the orientation and design of a webpage. It aligns the text in a container/element in a similar manner to a word processor.

    Common values:
    • - left (or start)
    • - right (or end)
    • - center
    • - justify
    • - justify-all

    Proper Usage of the text-align property:

      selector {
         text-align: center;
    }

    display

    display is one of the most important CSS properties. The way you set display determines the layout of a page, element, <div> or any other element for that matter. See more details about CSS Positioning with display and Flexbox at the link.

    display property values:
    • - block
    • - inline
    • - inline-block
    • - flex
    • - inline-flex
    • - grid
    • - subgrid

    Proper Usage of the display property:

      selector {
         display: flex;
    }

    position

    The CSS position property is most common in positioning the <header>. Looking at the relative and absolute values for display, relative takes into account content on the page and makes room. Absolulte doesn't include the selected element in the page flow, it just places it where it's told regardless of other content. The fixed value also removes it from the page flow and behaves like absolute, but it will stay on the page regardless of scrolling.

    • - relative
    • - absolute
    • - fixed
    • - sticky
    • - static

    Proper Usage of the position property:

      selector {
         position: fixed;
    }

    color

    The CSS color property is used for text. This property is used often when designing, and accepts values in a few ways. There are over 140 CSS (English) color inputs. You can input a rgb (number, number, number) value. You can input a rgba(number, number, number, index) value. You can input a hex value, like #bf00d1. You can also input a hsl or hsla value, do your research!

    • - English Color
    • - rgb(255, 255, 255)
    • - #000000
    • - hsl(%, %, %)
    • - rgba and hsla

    Proper Usage of the color property:

      selector {
         color: rgba(0, 0, 0, 0.5);
    }

    background-color

    The background-color property is used frequently and is similar to color.

    • - English Color
    • - rgb(255, 255, 255)
    • - #000000
    • - hsl(%, %, %)
    • - rgba and hsla

    Note that the a in rgba and hsla is the transparency value, between 0 and 1. Look at the top of this page. The title box has a background-color of
    rgba(255, 255, 255, 0.5).

    Proper Usage of the position property:

      selector {
         position: fixed;
    }

    background-image

    The background-image CSS property is used to put an image behind other elements. It is used on a large scale at the top of this webpage.
    • - url
    • - gradient

    The URL destination property can be a local (relative) or remote file path.

    Below, these are important. Other properties (recommended settings):

    • background-position: center;
    • background-size: cover;
    • background-repeat: no-repeat;

    Proper Usage of the background-image property:

      selector {
         background-image: url("https://mysite.io/cat.png");
    }

    width/height

    CSS width and height properties are used to size elements. They both take the same arguments and are usually used together. Sometimes only one needs to be declared, but it's always nice to have both.

    Aruments they take:
    • - 18vw (viewport width)
    • - 76vh (viewport height)
    • - 94px
    • - 100%
    • - 3rem

    Proper Usage of the width/height property:

      selector {
         width: 40vw;
         height: 25vh;
    }

    border

    The CSS border property is used to define the, well, border of an element! You can more specific and declare border-top, border-right, border-bottom, border-left, and make it curved with border-radius. The standard border syntax is below.

    Each of the three positions take the below arguments:
    • - First Argument: Width. Takes any sizing.
    • - Second Argument: Type. Takes a definition.
    • - Third Argument: Color. Takes any type of color.

    Second Argument types are: dotted, dashed, solid, double, groove, ridge, inset, outset, none, hidden.

    Proper Usage of the border property:

      selector {
         border: 3px solid blue;
    }

    text-decoration

    The text-decoration property is used to underline, overline, wavy line, get rid of line, you get the line. It has three arguments; text-decoration-line, text-decoration-style, and text-decoration-color. You can declare just the first one, like text-decoration: underline as well as different combinations of CSS shorthand.

    Some examples are:
    • - text-decoration: wavy overline purple;
    • - text-decoration: underline;
    • - text-decoration: none;
    • - text-decoration: dashed red;

    Proper Usage of the color property:

      selector {
         color: rgba(0, 0, 0, 0.5);
    }

    margin

    The CSS margin property is part of the Box Model, as well as padding, border, and width/height. More about the Box Model and Positioning in the next section of this guide. Margin is outside of an element between other elements.

    Syntax Variations:
    • - margin: top right bottom left;
    • - margin: top/bottom left/right;
    • - margin-top: 30px;
    • - margin: 2vh;

    Proper Usage of the margin property:

      selector {
         margin: 2vh auto;
    }

    padding

    CSS padding is similar to margin, but it's the space inside of an element. Also part of the Box Model.

    Syntax examples:
    • - padding: 3px;
    • - padding: 3vh 2vw;
    • - padding-left: 10px;

    Proper Usage of the padding property:

      selector {
         padding: 2vh;
    }

    max and min

    Using max and min properties is helpful when designing responsive webpages. There are times that you will want to have containers appear differently on different devices, and the difference isn't enough to justify a @media query.

    • - max-width: 30vw;
    • - min-height: 25vh;

    Proper Usage of the max/min property:

      selector {
         max-height: 400px;
    }

    @media

    Media queries are important in responsive web design. Their syntax is simple, and you place CSS code inside of a media query that you want to take effect when the query condition is met. Information in a media query overrides CSS calling outside of the media query when its condition is met.

    Syntax examples:
    • - @media only screen and (max-width: 480px) {}
    • - @media only screen and (min-resolution: 1440p {})
    • - @media only screen and (orientation: portrait) {}

    Proper Usage of the @media query:

      @media only screen and (max-width: 480px) {
         .home-content-container {
           max-width: 40vw;
      }
    }

    @font-face

    The @font-face query is an alternative to linking an external font in the <head>.

    @font-face has a few quirks that are different than @media.

    Proper Usage of the @font-face query:

      @font-face {
         font-family: "Open Sans";
         src: url("./fonts/OpenSans.woff2") format("woff2");
    }

    Note that the src line is usually on one line.

    line-height

    CSS line-height is used to give text more space between lines, as the name suggests. It takes a numerical value that is multiplied by the current line height. Common values for additional line-height are 1.2, 1.3, and 1.4.

    Syntax examples:
    • - line-height: 1.2;
    • - line-height: 1.2;

    Proper Usage of the line-height property:

      selector {
         line-height: 1.2;
    }

    Positioning

    The Box Model

    ***Image from Mozilla MDN***

         The Box Model is very important in developing webpages. By utilizing CSS properties like margin, padding, width, height, and border, you can position elements exactly where you want them.

         Margin is, as in the diagram to the left, space outside of an element that you can control to define distances between objects on a page. When set to a single number, margin adds that same amount on all sides. You can customize how margin is set individually for all sides, just for one side, for the top and bottom, or use filler values like auto to center in the direction you assign. See the above CSS section and read about Margin in its very own box!

         Padding is also important. It keeps your text and other elements from feeling "squished." The padding around each of the CSS boxes above is what keeps those elements from looking rough, compressed, ect. Utilizing padding gives your webpages more of a "controlled," or refined look. Asthetic is important because you want your visitors to feel comfortable; not struggling to find their way around your website.

         Border is the last property. It is most commonly used to literally, add a border! As with the other properties, see the boxes above for syntax and more!

    Flexbox Positioning

         Flexbox Positioning is a modern method that I use to lay out my webpages. I personally prefer it over Grid and using "Block" layout, but you need to know the basics of them all.

         The first important CSS property with Flexbox is display. Read about display above for syntax. When set to display: flex, an element behaves like a block. Flex can be aligned on two axis: the Main Axis and the Cross Axis.

         On the top diagram to the right, you can see that the main axis is lateral and the cross axis is vertical. The main axis is affected by the CSS properties flex-direction, and justify-content. The main CSS properties that affect the cross axis are flex-direction and align-items/align-content. To be clear, in order to declare one of these properties in CSS, you have to already have declared display: flex; in the element or a parent element. flex-wrap is an important element when laying out static galleries or containers in a row. You can set flex-wrap to wrap, no-wrap, or wrap-reverse. By allowing containers to wrap, you can make multi-row galleries that are responsive and can be adjusted for multi-size viewing. At the bottom of this webpage I'll provide links to parts of the Mozilla MDN and css-tricks.com for more material and self-help.

         Values for justify-content, align-items, and align-content are: center, flex-start, flex-end, space-between, space-around, and stretch. I encourage you to go to the bottom of the page and go visit my Source Code, and see some of these properties in action.

    Other Flex Properties:

    • - flex-basis
    • - flex-grow/flex-shrink

    Git and GitHub

        

         Git and GitHub together are very powerful tools. Together, you can successfully version and publish your code and even websites. This website is hosted with GitHub pages, free, and I can connect it to a domain of my choice.

    Common commands and defenitions in the Git Command Line Interface:

    • - "cd" _____ navigates to a file or file path that follows the command.
    • - "git init" initializes a Git directory in the current root file.
    • - "git status" checks the status of files in a git directory.
    • - "git add ." adds all files that have been modified to the staging area.
    • - "git commit -m "reason"" records modifications to the local repository.
    • - "git push" updates the files in your GitHub online database with local repository changes.


    Setup commands:
    1. 1. "git init"
    2. 2. "git add ."
    3. 3. "git commit -m "First Commit"
    4. 4. "git remote add origin git@github.com:User/UserRepo.git"
    5. 5. "git push -u origin master"

Git is a powerful tool that makes it easy and lets you say goodbye to the days of uploading folder by folder to some online "good deal" directory. Developers use GitHub for code projects in any language.
